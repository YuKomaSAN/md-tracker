# ウェブアプリ公開・共有手順書

このドキュメントでは、MD Trackerを他のユーザーも利用できるようにするための手順をまとめます。

## 1. 公開の仕組み

このアプリは「フロントエンド（画面）」と「バックエンド（データ保存）」に分かれています。

*   **フロントエンド (HTML/JS):** GitHub Pagesを使ってWeb上に公開します。誰でもブラウザからアクセスできるようになります。
*   **バックエンド (Google Sheets + GAS):** ユーザーごとに自分のGoogleアカウントでセットアップしてもらいます。
    *   *メリット:* ユーザーのデータが他人に漏れない、開発者のGoogleアカウントの制限（Quota）を気にしなくて良い。
    *   *デメリット:* ユーザーに最初のセットアップ（GASのコピペ等）をお願いする必要がある。

## 2. 実施手順

### Step 1: GitHub Pagesの設定 (フロントエンド公開)

リポジトリの設定を変更し、Webページとして公開します。

1.  GitHubリポジトリの **Settings** タブを開く。
2.  左メニューの **Pages** をクリック。
3.  **Build and deployment** > **Source** で `Deploy from a branch` を選択。
4.  **Branch** で `main` (または `master`) を選択し、フォルダは `/(root)` のまま **Save** を押す。
5.  数分待つと、ページ上部に `Your site is live at ...` とURLが表示されます。これがアプリのURLになります。

### Step 2: ユーザー向けマニュアルの整備

現在の `README.md` は開発者向けの情報が多いため、一般ユーザー向けの「セットアップガイド」を作成または追記する必要があります。

**必要な情報:**
1.  **GASスクリプトの準備:**
    *   `gas/コード.js` の中身をコピーしてもらう必要があります。
    *   一般ユーザーにはGitHubのファイル階層は分かりにくいため、コードを簡単にコピーできるページを用意するか、マニュアル内にリンクを目立つように貼ります。
2.  **GASのデプロイ手順:**
    *   スプレッドシート作成 -> 拡張機能 -> Apps Script -> コード貼り付け -> デプロイ -> URL取得 の流れを画像付きなどで分かりやすく説明します。
3.  **アプリの初期設定:**
    *   公開されたURLにアクセスし、取得したGASのURLを入力する手順を説明します。

### Step 3: 動作確認

1.  公開されたGitHub PagesのURLにアクセスする（シークレットウィンドウ推奨）。
2.  自分のGAS URLを入力して、正常に接続・記録できるか確認する。

## 3. 今後のアクションプラン

まずはこの手順に沿って進めますか？
よろしければ、以下の作業を私がサポートします。

1.  **ユーザー向けガイド (`USER_GUIDE.md`) の作成**: 専門用語を減らした分かりやすい手順書を作成します。
2.  **`README.md` の更新**: ユーザーガイドへのリンクを追加します。

GitHub Pagesの設定（Step 1）は、ユーザー様ご自身でブラウザ上のGitHub設定画面から行っていただく必要があります。
